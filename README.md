В проекте проведён продуктовый анализ маркетплейса Olist Store — бразильской e-commerce платформы, объединяющей множество продавцов и покупателей.
Использован общедоступный датасет, содержащий ~100 000 заказов за 2016–2018 годы с различных торговых площадок. Это реальные коммерческие данные, предварительно анонимизированные.

Язык и библиотеки:

Python
pandas, numpy
plotly, matplotlib, seaborn

Статистические методы:

chi-square test (chi2_contingency)
two-proportion z-test (proportions_ztest)
Mann–Whitney U test (mannwhitneyu)
доверительные интервалы для долей (proportion_confint)

В проекте из сырых таблиц была собрана аналитическая модель, состоящая из двух ключевых витрин:
item_fact — витрина на уровне товара

Содержит детализированную информацию по каждой позиции в заказе:

Что включает:

данные о товаре (категория, вес, размеры)
стоимость товара и доставки
продавца
дату покупки и доставки
регион клиента
статус доставки

Дополнительно рассчитано:

item_total_value = price + freight_value
product_weight_kg
volume_cm3
флаг is_delivered

order_fact — витрина на уровне заказа

Агрегированная витрина, объединяющая всю информацию по заказу:

Что включает:

данные заказа и клиента

количество товаров (items_count)
количество продавцов (sellers_count)
сумма заказа (order_items_total_value)

Ключевые расчетные метрики:

is_late = delivery_days_actual > delivery_days_estimated

В рамках проекта было проверено несколько гипотез о причинах просрочек доставки.
Ключевыми оказались две:

Гипотеза 1: География влияет на просрочки доставки

Идея:
разные регионы могут иметь разное качество логистики → это влияет на SLA.

Как проверяли:

рассчитали долю просрочек по штатам
построили карту
применили chi-square test
рассчитали доверительные интервалы

Результат:

выявлена статистически значимая зависимость между регионом и просрочкой (p-value ≈ 0)
отдельные штаты стабильно показывают более высокий уровень задержек

👉 Вывод:
логистика неоднородна → проблема системная, а не случайная

Гипотеза 2: Нарушение сроков продавцом влияет на итоговую доставку

Идея:
если продавец поздно передаёт товар в доставку, заказ с большей вероятностью опоздает.

Как считали:

seller_delay_days = carrier_date - shipping_limit_date
seller_late = seller_delay_days > 0

Как проверяли:

сравнили доли просрочек (z-test)
сравнили время доставки (Mann–Whitney U test)

Результат:

доля просрочек растёт с 6.4% до 32.7%
время доставки почти удваивается
различия статистически значимы

👉 Вывод:
это ключевой драйвер плохого пользовательского опыта

Когортный анализ и удержание

Был проведён когортный анализ пользователей по месяцу первой покупки.

Что считалось:

retention по месяцам
repeat rate
влияние первой доставки на возврат

Ключевые результаты:

только ~3% пользователей совершают повторную покупку
~97% не возвращаются

Самый важный инсайт:

👉 качество первой доставки критично влияет на удержание:

доставка вовремя → 5.0% retention на 1 месяц
доставка с просрочкой → 0.5% retention

➡️ разница в 10 раз
